# Rutibooki 인터리빙 모드 설계 회의록

- **날짜**: 2026-04-18 (저녁 세션)
- **참석자**: 이교수(독서교육 전문가, 15년차 / 정독주의), 최박사(심리학·행동심리 박사 / 습관·선택 피로 연구), 김선배(10년차 프론트엔드 개발자 / 실용주의, 1차 회의 연속 참석)
- **주제**: 인터리빙 모드의 개념·분량·순서·UX·데이터 연속성·실패 시나리오

## 0. 회의 시작 — "애초에 이게 왜 다시 올라왔냐"

**김선배**: 1차 회의(오전)에서 인터리빙은 "MVP에서 빼자"로 합의했는데, 혁준님이 이미 루틴 ↔ 인터리빙 **토글 UI를 붙여버렸음**. 이제 뺄 수 없으니까 설계하자는 거.

**최박사**: 순서가 꼬인 건데, 오히려 이게 기회일 수도. 토글이라는 건 **"둘 다 내 선택지"**라는 메시지라서, 사용자에게 모드 전환 자체가 하나의 **변동성(variability)** 을 제공함. Zeigarnik 효과 — 완결되지 않은 것에 끌리는 심리 — 관점에선 나쁘지 않음.

**이교수**: 저는 반대로 시작합니다. 인터리빙(Interleaving)은 Rohrer & Taylor(2007)의 수학 학습 실험에서 나온 개념이고, **"변별력"을 기르는 게 목적**이에요. 독서는 변별이 목적이 아니라 **몰입(flow)이 목적**입니다. Csíkszentmihályi 플로우 이론에서 몰입은 **맥락 전환 비용(context switching cost)** 에 가장 취약한데, 책을 섞으면 그 비용이 챕터마다 발생해요.

**김선배**: 교수님 말씀은 알겠는데, Rutibooki의 북극성은 "매일 책 펼치기"지 "깊게 읽기"가 아님. 오전 회의에서 확정된 거.

**이교수**: 바로 그 지점이 우려스럽습니다. **"펼침"을 최적화하다가 "읽기"를 잃는** 구조가 될 수 있어요.

## 1. 인터리빙의 개념 — 클러스터 내 vs 클러스터 간

**이교수**: 방향①(클러스터 내 교차, 예: 돈·투자 3권을 챕터별로 섞음)이 **학습 이론상 정답**입니다. Bjork(1994)의 "desirable difficulty" — 같은 주제를 다른 저자로 읽으면 개념이 추상화돼서 전이(transfer)가 일어나요. "불변의 법칙"의 복리 개념과 "돈의 방정식"의 복리 개념이 부딪히면서 **메타 이해**가 생깁니다.

**최박사**: 이론상 맞는데 현실은 다릅니다. 같은 주제 3권 연달아 읽으면 **"또 돈 얘기야?"** 피로가 쌓여요. Ariely의 선택 피로 연구처럼, 인지 자원이 비슷한 영역에서 연속 소모되면 이탈률이 급등합니다. 저는 방향②(클러스터 간 교차)를 지지해요. 돈 → 습관 → 인간본성 → 돈 순으로 가면 **뇌가 환기**되고, 변동 보상(variable reward) 구조가 만들어집니다. 닐 이얄의 Hook 모델에서 "variable reward"는 재방문을 만드는 가장 강력한 훅이에요.

**이교수**: 근데 박사님, 그건 **독서가 아니라 스낵**이에요. 클러스터 간 교차는 책 7권을 "피드"로 만드는 겁니다. 틱톡 원리.

**최박사**: 틱톡 원리가 나쁘다고만 볼 수 없어요. 북극성이 "매일 펼침"이면 틱톡 원리가 **오히려 북극성에 가까워요**.

**김선배**: 중재하자면 — 기술적으론 **두 개 다 구현 가능**함. `interleaveStrategy: 'within-cluster' | 'across-cluster'` 플래그 하나만 있으면 됨. 문제는 디폴트를 뭘로 두냐.

**이교수**: 디폴트는 ②(클러스터 간)로 가더라도, ①(클러스터 내)을 **"정독 모드"로 네이밍**해서 드러내는 걸 제안합니다. 사용자가 "아 나 지금 깊게 읽고 싶어" 할 때 고를 수 있게.

**최박사**: 네이밍은 찬성. 근데 "정독 모드"라는 단어가 이미 부담을 줘요. **"같은 주제 묶어 읽기"** 처럼 서술형으로.

> 🔴 **혁준님께 결정 1**
> 인터리빙 디폴트 전략을 **(A) 클러스터 간 교차 — 돈→습관→본성→돈 (뇌 환기 우선)**, **(B) 클러스터 내 교차 — 돈 3권만 섞어 (정독 전이 우선)**, **(C) 둘 다 제공하고 사용자가 고름 (첫 진입 때 1회)** 중 뭐?

## 2. 하루 분량 — 몇 권, 몇 챕터?

**최박사**: 하루 **2권 × 1챕터씩**이 상한선입니다. BJ Fogg의 Tiny Habits 원칙 — 습관 형성기엔 "바보처럼 작게". 3권 이상 넘기면 "이걸 언제 다 해" 감정이 와요.

**이교수**: 2권도 많아요. 챕터 하나 깊이 읽으면 여운이 필요합니다. **"1권 1챕터 + 선택적 보너스 1챕터"** 를 제안합니다.

**김선배**: 데이터 관점으로 — 챕터 분량이 책마다 천차만별임. "훅"의 한 챕터는 10분, "사피엔스"의 한 챕터는 40분. **"챕터 수"가 아니라 "예상 독서 시간"으로 묶어야** 공정함. 대략 **하루 20~30분 분량**을 목표로 2~3조각으로 쪼개기.

**최박사**: 동의. 단, 시간 표기는 **보이지 않게**. 타이머 강조하면 오전 회의 결론(타이머 구석으로)과 충돌.

**이교수**: 그럼 UI엔 "오늘의 독서 3조각" 정도로 추상화하고, 내부 스케줄러만 시간 기반으로 계산하는 게 맞습니다.

**합의**: 하루 **2~3개 챕터 카드**, 내부 계산은 **25분 타깃**, UI엔 시간 노출 X.

## 3. 순서 — 사용자가 고르냐, 알고리즘이 정하냐

**최박사**: 오전 회의 핵심이 **"선택을 대신 해주는 앱"** 이었죠. 순서도 앱이 정해야 합니다. 선택지를 주면 루틴 모드의 장점(요일별 고정으로 결정 제거)이 무너져요.

**이교수**: 저는 "오늘의 3조각"을 **보여주되, 순서는 자유**로 두는 걸 선호합니다. 독자에게 **최소한의 자율성**은 남겨야 해요. Deci & Ryan의 자기결정성 이론(SDT) — autonomy가 없으면 내재 동기가 죽습니다.

**김선배**: 개발자 입장 — 순서를 고정하면 "다음 챕터 자동 재생" 같은 **플레이리스트 UX**가 됨. 순서를 자유로 두면 **덱 카드 UX**가 됨. 둘이 완전히 다른 컴포넌트라서 여기서 결정이 되어야 구현 들어감.

**최박사**: 절충안 — **순서는 알고리즘이 제안하되, 드래그로 재정렬 가능**. 디폴트에 95% 순응할 거고, 한두 번 바꾸는 행위 자체가 **소유감(ownership)** 을 만듭니다. IKEA 효과.

**이교수**: 그게 좋겠네요. "추천 순서"라는 라벨 달고.

## 4. 인터리빙 화면 UX

**김선배**: 제가 후보 3개 뽑아왔어요.

```
[후보 A — 플레이리스트 카드]
┌─────────────────────────┐
│ 오늘의 독서 3조각         │
│                         │
│ 1. 💰 불변의 법칙         │
│    5장: 꼬리의 힘          │
│    ▸ 시작                │
│ ─────────────────────    │
│ 2. 🧠 훅                 │
│    3장: 변동 보상          │
│    ▸ 시작                │
│ ─────────────────────    │
│ 3. 🦴 완벽한 원시인       │
│    2장: 사냥꾼의 주의력     │
│    ▸ 시작                │
└─────────────────────────┘
```

```
[후보 B — 덱 스와이프]
     ┌──────────┐
     │  💰       │
     │ 불변의    │  ← 현재
     │ 법칙      │
     │ 5장       │
     └──────────┘
      ↑ 스와이프 ↓
```

```
[후보 C — 정류장 변형]
 💰 ──── 🧠 ──── 🦴
 불변    훅      원시인
 ●       ○       ○
 지금    다음    그 다음
```

**이교수**: A(플레이리스트)가 가장 **"읽기 모드"에 가깝습니다**. B(덱)는 틱톡/인스타 릴스 메타포라서, 사용자가 "휙휙 넘기는 행동"을 학습해요. 첫 챕터도 안 읽고 스와이프할 가능성 큽니다.

**최박사**: 저도 B는 위험해요. 덱 UX는 **거부가 기본값**인 인터랙션입니다(틴더). 독서는 거부가 기본이면 안 돼요. A 지지.

**김선배**: 합의 쉽네요. **A(플레이리스트)** 로 갑시다. C(정류장)는 오전 회의에서 루틴 모드에 배정된 UI라서, 여기서 또 쓰면 두 모드 구분이 흐려짐.

**최박사**: 단, A에서 **카드 간 구분선을 뚜렷하게**. 연속 스크롤 피드처럼 보이면 틱톡화됩니다. 카드 사이에 **"한 템포 쉬어가는 여백"**.

**이교수**: 그리고 카드 안에 **"이 챕터를 읽는 이유"** 한 줄. 예: "지난 주 '돈의 방정식' 3장과 이어지는 개념". 맥락을 주지 않으면 인터리빙이 **그냥 랜덤 재생**이 됩니다.

## 5. 루틴 ↔ 인터리빙 전환 시 데이터 연속성

**김선배**: 이게 가장 조용한 지뢰임. 루틴 모드에서 월요일에 "불변의 법칙 5장"을 읽다가 3페이지까지 갔음. 화요일 오전에 인터리빙 모드로 토글. 인터리빙 스케줄러는 "불변의 법칙 5장"을 또 오늘에 넣을 수도, 안 넣을 수도 있음. **진행도는 어디에 저장되냐?**

**최박사**: 진행도는 **책×챕터 단위로 단일 저장**해야 합니다. 모드는 "보여주는 방식"일 뿐이지, 데이터의 소유자가 아니에요.

**이교수**: 동의. 책은 하나, 진행도 하나. 모드는 **렌즈**일 뿐.

**김선배**: 스키마 제안 (Rutibooki 컨벤션 — TypeScript).

```ts
// src/types/reading.ts
type ChapterProgress = {
  bookId: string;
  chapterIndex: number;
  status: 'unread' | 'in-progress' | 'done';
  lastOpenedAt: string; // ISO
  // 모드와 무관. 어느 모드에서 열었든 여기에만 기록.
};

type DailyPlan = {
  date: string; // YYYY-MM-DD
  mode: 'routine' | 'interleave';
  cards: Array<{ bookId: string; chapterIndex: number }>;
  // 모드별 plan은 분리. 진행도는 ChapterProgress가 단일 소스.
};
```

**최박사**: 모드 전환 시 **"어제까지 읽던 챕터"를 오늘 카드 1번에 고정** 시키는 규칙 하나만 넣으면 사용자 혼란 대부분 해소됩니다. "어디 있었지?" 감정이 안 옴.

**이교수**: 중요한 포인트네요. "섞어 읽기"라도 **미완결 챕터를 내버려두지 않는다**는 안전장치.

**김선배**: `generateInterleavePlan()` 첫 줄에 "미완 챕터 있으면 최우선 삽입" 로직. 복잡도 낮음.

## 6. 실패 시나리오 — 첫 챕터만 읽고 이탈

**최박사**: 인터리빙에서 가장 흔한 실패 패턴입니다. 3조각 중 1개만 완료 → 죄책감 → 내일 앱 안 엶. Ovsiankina 효과(미완결 과제 재개 욕구)가 독서에선 **역으로 작용**해요. "다 못 끝낸 오늘"이 스트레스원이 되면서.

**이교수**: 그럼 해결은 간단합니다. **하루 완료 기준을 "3조각 전부"가 아니라 "1조각 이상"으로**. 나머지는 "보너스"로 표기. 오전 회의 북극성이 "펼침률"이면 1조각으로 이미 성공입니다.

**최박사**: 완전 동의. 카드 UI에:

```
✅ 오늘 독서 완료  (1/3)
   나머지 2개는 보너스예요 — 내일로 넘겨도 OK
```

이 **"넘겨도 OK"** 한 줄이 앱의 톤을 결정합니다.

**김선배**: 구현 관점 — 안 읽은 카드를 다음날로 **자동 이월**할지, **폐기**할지 결정 필요. 이월하면 스케줄이 점점 뚱뚱해짐. 폐기하면 섭섭함.

**최박사**: **폐기 + "지난 주 놓친 챕터" 섹션을 주간 회고에 한 번**. 매일 쌓이면 부담, 주 1회면 재발견 기쁨.

**이교수**: 훌륭합니다. **"이월 대신 재발견"** — 이거 메모해 두세요.

## 합의 사항 (3인 공통)

- 인터리빙 화면은 **플레이리스트 카드(후보 A)**. 덱 스와이프(B)는 틱톡화 위험으로 폐기.
- 하루 **2~3개 챕터 카드**, UI엔 시간 표기 X, 내부 계산만 **25분 타깃**.
- 진행도는 **`ChapterProgress` 단일 소스**, 모드는 렌즈일 뿐.
- 모드 전환 시 **미완결 챕터를 오늘 카드 1번에 자동 고정** (혼란 방지 안전장치).
- 카드 사이 **여백 넉넉히** — 연속 스크롤 피드처럼 보이면 안 됨.
- "오늘 완료" 기준은 **낮게, 보너스 개념으로 여유**를 둔다 (정확한 숫자는 결정 3에서 확정).
- 루틴 ↔ 인터리빙 **둘 다 동등한 시민**. 한쪽을 "심화 모드"로 숨기지 않음 (오전 합의에서 업데이트).
