**일시**: 2026-04-18 ~ 04-19
**참석자**: 혁준, 이교수(독서·인지심리), 김개발(프론트엔드), Nir Eyal(*Hooked*/*Indistractable*), Maryanne Wolf(*Reader, Come Home*), Andrew Huberman(*Huberman Lab*)

## 📜 앱의 헌법 (추구미)

> **"오늘도 책을 읽은 사람에 합류했다. 조금이라도 괜찮다.
> 누군가 책을 볼 때면 항상 켜져있고 싶은 앱. 조용한 도움을 주는 앱."**

모든 구현·디자인 결정은 이 문장을 통과해야 한다.

핵심 전제:
- **타겟**: 마음먹었지만 스스로 통제가 어려운 사람
- **태도**: 강제 금지, 조용한 도움
- **복귀 방식**: 책이 읽고 싶을 때 자발적으로 찾아오게

## ✅ 확정 결정사항

### A. 딴짓 방지 모드

**철학**: 강제하지 않는다. 하지만 책 읽는 데는 무조건 도움이 된다.

- **Default ON** — 앱 소개가 이미 암묵적 동의
- **실제 차단 X** — iOS PWA는 기술적으로 앱 이탈 막을 수 없음
- **복귀 시 "죄책감 UI"** — `visibilitychange` 감지
  - 앱을 떠났다 돌아오면 정류장 화면 + "잠깐 다녀오셨네요 📖"
  - 비난조 절대 금지
- **두 버튼만 제시**:
  - `[계속 읽기]` 초록, 즉시 탭
  - `[오늘은 그만]` 빨강, long-press 0.8초
- **색 대비 규칙**: 계속/그만 두 버튼 모두 빨강 금지 — 심리적 바다 피하기

### B. Reading 화면 (책 읽는 동안)

- **완전 검정 100%**, 1px 선도 없음
- **Wake Lock API** 지원 시 화면 꺼짐 방지 (iOS 17+, Android)
- **터치 시 정류장 표시** (검정 ↔ 정류장 토글)
  - 잠깐 멈춤 신호 — 종이책에서 손가락 끼우는 것과 같은 제스처
- **2분 간격 자동 저장** — Wake Lock 실패 시 안전망
- **Long-press 0.8초**로만 종료 (1.5초는 너무 길다고 결정)
- **첫 세션 1회 코치마크**: *"화면이 어두워져요. 다 읽으면 꾸욱 눌러 종료."*

### C. 시간 프레이밍

- **Pre-reading**: "보통 이 분량은 18분 정도 걸려요" (anchor만, 목표 아님)
- **Reading 중**: 시간 정보 완전 숨김
- **Post-reading**: 걸린 시간 노출 금지, 완료 확인만
- **프레이밍 원칙**:
  - 목적: "너 이 정도 시간이면 끝낼 수 있어, 시간만 내!"
  - 절대 피함: "18분 안에 끝내야 해!" (압박)

### D. Streak — 완전 제거

- **이유**: dopamine hook이 독서의 내재 보상을 오염시킴
- **Routinery와 차이**: 그 앱은 '흐트러진 유저 잡기' 목적, 우리는 '결심한 사람 존중'
- **끊김 알림도 없음** — 이분법 자체를 제거
- **대신**: 설정 메뉴의 **"내 독서 지도"** — 월간 히트맵
  - GitHub 잔디밭 스타일, 색의 진하기만 표시, 숫자 없음
  - 유저가 능동 조회 (push 아님, pull)

### E. 온보딩 — 과학 카드 3장

스와이프 넘기기, 각 카드에 논문 레퍼런스 작게. 스킵 즉시 가능.

**카드 1**
> "같은 시간대에 읽을 때 뇌의 기억 회로가 더 잘 연결돼요."
> *— Circadian encoding, Chavan et al. 2016*

**카드 2**
> "스마트폰 옆에 두지 않으면 집중력이 26% 올라요."
> *— Ward et al. 2017, Brain Drain study*

**카드 3**
> "읽은 직후 10초만 가만히 있으면 기억이 두 배로 오래가요."
> *— Dewar et al. 2012, wakeful rest*

각 카드 끝: *"이 원칙을 어떻게 쓸지는 당신이 정하세요"*

**시간대 질문 없음** — 앱이 조용히 관찰해서 히트맵에만 반영.

### F. 인용 필드 — 파트 완료 시점

- **매일 X** (부담), **완독 X** (너무 멈)
- **파트 완료 시점** — 유저 입력 페이지가 파트 경계를 넘으면 자동 모달
- **문구**: "이 파트에서 남기고 싶은 한 문장 (선택)"
- **비워도 확인 가능** — 설득 없음
- **인터리빙 모드에도 동일** — 각 책의 파트 경계 기준

**왜 파트인가**:
- 파트 경계 = 뇌의 episodic boundary (의미 통합 지점)
- 너무 자주도 너무 가끔도 아닌 적정 주기
- 유저가 자연스럽게 '뭔가 남기고 싶은' 감정 타이밍

### G. 정류장 애니메이션

- **노출 3곳**:
  1. 홈 큰 책 카드뒤집기 (능동 조회)
  2. 딴짓 복귀 화면 (상황 발생)
  3. Post-reading 이동 애니메이션 (하루 1회 보상)
- **애니메이션 스펙**:
  - 0~0.3s: 현재 점 squash (움츠림)
  - 0.3~1.2s: 다음 정류장으로 스프링 이동 (오버슈팅 easing)
  - 1.2~1.8s: 도착점 3번 펄스 (stretch)
  - 1.8~2s: 경로 초록 채워짐
  - 파트 경계 넘으면 +2~2.5s 글로우 + 진동
- **무음 유지**

### H. Post-reading 완료 문구 (고정, 매일 동일)

```
오늘도 책을 읽은 사람에 합류했어요.
조금이라도 괜찮아요.
```

- 변주·개인화 금지 (의식성 보존)
- 3초 후 자동 홈 복귀

### I. 페이지 입력 pre-fill

- **기본값**: 오늘의 section 끝 페이지
- **EMA 가동 시점**: 3회 누적 이후
- **입력 UX**:
  - 숫자 키패드 자동 호출 (`<input type="tel">`)
  - "확인"만 눌러도 기본값으로 진행
  - 덜 읽었으면 수동 수정

### J. 모드 구조

- **MVP = "한 권 모드" 단일 집중**
- 요일 루틴, 인터리빙은 V2 스코프
- **모드 선택 UI 없음** — 서재 구성으로 자동 결정
- 책 쌓이면 조용한 힌트만: *"책이 3권 쌓였어요. 요일별로 나눠 읽어볼까요?"* (제안일 뿐)

### K. PWA 설정

- **MVP 배포 전 30분 작업으로 완료**
- 필요 파일:
  - `public/manifest.json`
  - `public/apple-touch-icon.png` (180×180, icon.svg에서 변환)
  - `layout.tsx` meta 태그 추가
- 결과: iOS "홈 화면에 추가" 시 브라우저 주소창·메뉴 제거, 네이티브 앱처럼 동작

## 🎬 최종 확정 플로우

```
[홈 화면]
  - 오늘의 책 표지 (탭 → 카드뒤집기로 정류장)
  - 오늘 분량 카드 "12~28p · 보통 이 분량은 18분 정도"
  - [책 펼쳤어요 →]
       ↓
① Pre-reading (3초 페이드)
   "파트 4 · 12~28p"
       ↓
② Reading (완전 검정)
   - 100% 블랙
   - 터치 시 → 정류장 토글 (잠깐 멈춤)
   - Long-press 0.8초 → 종료
   - 2분 간격 자동 저장
   - 앱 이탈 감지 시 복귀 화면 준비
       ↓
  [만약 앱 이탈 후 복귀]
   → 정류장 + "잠깐 다녀오셨네요 📖"
   → [계속 읽기 초록] [오늘은 그만 빨강 long-press]
       ↓
③ Post-reading 입력
   "어디까지 읽었어요?"
   [ 28 ] ← pre-fill (section 끝 페이지)
   [ 확인 ]
       ↓
④ 정류장 스프링 이동 애니메이션 (2초)
   (파트 경계 넘으면 +글로우 연출)
       ↓
  [파트 경계 넘은 경우에만]
   → 모달: "이 파트에서 남기고 싶은 한 문장 (선택)"
   → 비워도 OK
       ↓
⑤ 완료 화면 (3초 후 자동 홈 복귀)
   "오늘도 책을 읽은 사람에 합류했어요.
    조금이라도 괜찮아요."
```

## 🔬 적용된 이론적 근거

- **Consolidation window** (이교수) — post-reading 10~15초 조용히 유지
- **Hooked 모델 재해석** (Nir) — Variable Reward는 정류장만으로 충분, Streak 제외
- **Deep reading** (Wolf) — 화면 검정, 방해 없음, 선택적 인용
- **Dopamine 회로 오염 방지** (Huberman) — 숫자 보상 제거, 사회적 보상(합류)로 전환
- **Basal ganglia 시간 인코딩** (Huberman) — 시간대 질문은 안 하되 앱이 조용히 학습
